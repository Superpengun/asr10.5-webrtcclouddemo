1. 序言

本文档介绍灵云 SDK Java 版的组成、功能以及如何使用 Java 语言开发灵云 SDK 应用， 预期读者为灵云 SDK 应用开发者。本文档中不包含 API 参考，请使用灵云 SDK Java 开发手册查询具体 API。

2. 概述

灵云 SDK Java 版面向 Windows, Linux 和 Android 三个系统，核心功能使用 C++ 语言开发实现，通过 JNI 封装为 Java 版开发包。灵云 SDK 主要用于人机交互场景， 应用中多使用异步处理。因此灵云 SDK 提供的功能接口多为异步接口，同时也提供必要的同步形式接口， 以满足用户开发简单应用的需求。

2.1 开发包文件

灵云 SDK 目前支持六个平台架构:

  • Linux 系统: 支持 x86_64, x86 架构
  • Windows 系统: 支持 x64, x86 架构
  • Android 系统: 支持 arm64-v8a, armeabi-v7a 架构

每个系统对应一个开发包文件，包含该系统下的各架构开发所需文件。Android 开发包仅支持 Java 开发语言，其他开发包则均同时支持 Java 和 C++ 开发语言。

每个开发包中的文档、头文件、jar 包是系统无关的，在各系统之间不存在差异。

Android 系统开发包不含 C++ 语言头文件，但用户可从 Linux 或 Windows 系统开发包中提取 C++ 语言头文件，并使用 Android 系统开发包中的库文件开发 Android Native 库或应用。

2.2 模块划分

灵云 SDK Java 版主要由系统模块和具体的能力模块组成，当前版本能力模块仅有 ASR 模块 和 TTS 模块。C++ 通用模块中的功能封装在通用模块中。

3. 系统模块

灵云 SDK 系统模块提供网络访问、事件循环、日志输出、定时器、授权(功能暂缺)、音频流 (音频源和音频槽)等基础功能。部分功能仅面向能力模块，为能力模块实现业务功能提供支持。

3.1 网络访问

Java 版中网络访问功能未开放给开发者。

3.2 事件循环

一般情况下，异步回调会发生在事件循环线程中。开发者在回调中不应执行需长时间等待的阻塞式操作， 否则可能会影响灵云 SDK 的正常运行。

3.3 日志输出

初始化 HciSdk 时设置了 HciSdkConfig.data_path 字段，即会输出日志至 hci.log 文件中。文件保存在该字段指定的目录下面，当日志大小超出一定大小时，会进行日志文件滚动操作， 将较老的日志移动至 hci.log.1, hci.log.2,... 等文件中。用户可通过 HciSdk.setLogLevel 修改日志级别，以控制哪些级别的日志可以输出至日志文件。

为了方便用户在测试阶段使用日志，可在应用启动前设置环境变量 CONSOLE_LOG 为任意内容，这样 HciSdk 会同时将日志输出控制台(应用的标准错误输出)。在 Android 平台下，不支持 CONSOLE_LOG 功能， 但输出至日志文件的内容总会输出至 LogCat 日志中。

不建议在正式产品中启用 CONSOLE_LOG

3.4 定时器

使用 HciSdk.newTimer 方法创建定时器，可以设置定时器在指定时间后触发，或者以指定的间隔进行周期性触发。

3.5 授权功能

授权功能仅针对本地能力，当前版本中暂无本地能力；云端能力由云端控制授权。

3.6 音频流

在和音频数据打交道的能力场景中，有时会使用音频流，主要有两个接口:

  • 音频源接口。 以数据流的方式，按指定参数格式读取音频数据。
  • 音频槽接口。 以数据流的方式，按指定参数格式写入音频数据。

一些音频相关的能力场景中，会涉及端点检测 (VAD)，检查音频是语音还是静音，判断出语音段的起始及结束端点。 做完端点检测后，一个音频流将被分割成语音和静音交替出现的多段数据。

用户在音频源读取启动后可通过生效的音频参数设置中的 have_vad 字段，判断读取的音频数据是否包含端点检测信息。 如果 have_vad 为 true 的话，读取操作在分段交界处会返回 HciAudioError.VOICE_START 或 HciAudioError.VOICE_END 事件; 否则读取操作不会返回这两个事件。

用户在音频槽接口启动写入时，可以设定 have_vad 为 true，以表示写入的音频流已做端点检测处理。 之后用户应使用 IAudioSink.voiceStart() 和 IAudioSink.voiceEnd() 写入端点信息， 从而将写入的音频流分割为多个语音/静音段。

当前 SDK 功能实现中，用户可接触到的接口中还未用到端点检测，因此用户不需要设置 have_vad 或处理 have_vad 字段

3.6.1 音频源

用户有三种方式创建音频源对象：

  • 基于内存数据创建音频源。请参考 DataAudioSource 类
  • 基于音频数据文件创建音频源。请参考 FileAudioSource 类
  • 基于音频缓冲区创建音频源，需要用户主动向音频缓冲区的音频槽接口写入音频数据。请参考 HciAudioBuffer 类

前两种方式适用于简单的音频数据源，使用音频缓冲适用其他更为复杂的音频源场景。

3.6.2 音频槽

在 SDK 当前版本支持的场景中仅会在使用音频缓冲区时会用到音频槽。

4. 能力模块

每个具体的能力均有对应的模块，根据能力的应用场景，每个能力模块中至少会提供一个能力会话类。 能力会话类均继承自 HciSession 类。 能力会话类一般会同时支持云端能力和本地能力。创建能力会话时，用户应给出不同类型的配置， 以决定使用云端能力还是本地能力。

目前只有云端能力

4.1 ASR 模块

支持四种场景的识别模式。

  • 一句话识别

    一次性传入全部音频数据进行识别，适用于对话聊天，控制命令，输入法等识别较短语音的场景。

  • 流式一句话

    本模式下会话将用户提供的音频数据分割成多个分片发送至云端或送入本地识别引擎进行识别， 引擎在会话启动后即开始工作，随时对新送入的数据进行识别，无须等待所有音频数据就绪再开始， 因此响应时间短，用户体验好。本模式下不做端点检查，因此不进行断句，无法给出语音开始和结束事件。 一般用于较短的一句话识别，例如：手机输入法、智能家居语音交互场景。

  • 实时转写连续模式

    本模式下同样将音频数据分割成多个分片发送至云端或送入本地引擎进行识别，在识别之前先进行端点检测， 如果是语音才进行实际解码工作，否则将直接丢弃。检测到一段语音的结束点时，会将当前这段语音的识别结果返回； 然后继续检测后面的数据。在本模式下，可能会返回多次识别结果。本模式适用于长时间连续语音的实时转写， 例如：会议转写系统。

  • 实时转写单句模式

    本模式与实时转写连续模式类似，不同之处在于返回第一段识别结果后，不再处理后续的音频，会话自动结束。 这主要是因为在和用户进行语音交互的场景下，当用户说完一句话后，往往会等待后续的交互操作， 例如聆听根据识别结果播报的相关内容，因而没有必要继续识别后续的音频。

云端接口有 Http 和 WebSocket 两种形式，与其对应的能力会话分别是 FreetalkShortAudio 和 FreetalkStream。

FreetalkShortAudio 支持一句话识别，FreetalkStream 支持流式一句话，实时转写连续模式和实时转写单句模式。

4.2 TTS 模块

支持两种语音合成。

  • 短文本语音合成

    输入一段文本，语音合成完成后一次性返回合成的音频。云端能力通过 HTTP 接口实现。适合无明确播放延迟的使用场景。

  • 流式语音合成

    输入一段文本，将合成的音频分片持续返回，云端能力通过 WebSocket 接口实现。适合对播放延迟有较高要求的使用场景。

5.2.1 初始化

用户应设置正确的 appkey, secret, sys_url 和 cap_url 进行初始化。如果仅使用本地能力， 可以不设置 sys_url 和 cap_url。

HciSdk sdk = new HciSdk();
HciSdkConfig cfg = new HciSdkConfig();

cfg.setAppkey("aicp_app");  // 平台为应用分配的 appkey
cfg.setSecret("QWxhZGRpbjpvcGVuIHNlc2FtZQ");  // 应用对象的密钥 (敏感信息，请勿公开)
cfg.setSysUrl("https://10.1.18.101:22801/");
cfg.setCapUrl("http://10.1.18.101:22800/");
cfg.setDataPath(data_path);  // 数据存储目录
cfg.setVerifySSL(false);

Log.d("sdk-config", cfg.toString()); // 输出初始化配置至 LogCat

sdk.setLogLevel(LogLevel.D); // 调试阶段，日志级别设为 D。默认为 I

sdk.init(cfg, context);  // 初始化 HciSdk

在 Android 应用中，调用 HciSdk.init 传入的 context 应为应用的 Activity。根据设置参数的不同，用户应在 Android 应用中启用相应的使用权限(uses-permission)。

  • 设置 data_path 字段，需要启用 android.permission.WRITE_EXTERNAL_STORAGE 权限
  • 设置 sys_url 字段，需要启用 android.permission.ACCESS_NETWORK_STATE 和 android.permission.INTERNET 权限

灵云 SDK 初始化时若检查到没有所需的使用权限，将会输出错误日志并令应用闪退。当应用发生闪退时， 请仔细检查日志中的错误信息。

在其他平台下，context 参数无意义，可传入 null 值。

5.2.1.1 SSL 证书

灵云平台系统服务接口必须使用 https 访问(上面示例代码中的 https://10.1.18.101:22801)， 能力服务接口则可以使用 http 或 https (上面示例代码中的 http://10.1.18.101:22800)。

出于安全性的考虑，在生产环境中上述 URL 应使用域名代替 IP 地址，能力服务接口也应使用 https 连接，以保护会话令牌及数据的安全。通过 cfg.setVerifySSL(false)，SDK 将不验证服务接口证书的有效性，不应在生产环境中使用该配置。

如果 SSL 服务器证书是通过权威证书机构申请的合法证书，SDK 一般无须做额外的操作， 因为大部分系统中已预置安装了常见权威证书机构的根证书， SDK 可通过根证书验证根证书直接或间接颁发的服务器证书的合法性。

如果服务器证书是企业自己制作的自签名证书或者系统中不包含服务器证书的根证书， 则需要使用 setCaBundle 方法添加受信任的证书链。

以 https://baidu.com 为例，可通过下述方法获取证书链。

  1. 在 Chrome 浏览器中打开 https://baidu.com 首页，点击地址栏左侧的锁形图标并查看证书
  2. 在证书路径标签页，可以看到
    GlobalSign Root CA - R1
└── GlobalSign Organization Validation CA - SHA256 - G2
    └──baidu.com
  3. 选择 GlobalSign Root CA - R1 节点，点击 查看证书 按钮
    • 在详细信息标签页，点击复制到文件
    • 在证书导出向导中选择保存为 Base64 编码 X.509 (.CER) 格式的一个文件
  4. 使用同样的方式导出 GlobalSign Organization Validation CA - SHA256 - G2 证书
  5. 将两个证书文件合并成一个文件，按证书路径中的先后顺序排列，得到证书链文件 ca.pem 如下

5.2.2 反初始化

调用 HciSdk.close() 方法进行反初始化，该方法是一个异步方法。

同步方式

sdk.close();
sdk.waitForClosed();  // 等待 HciSdk 完成关闭操作，不能在事件循环线程调用
... // 资源释放等清场操作

异步方式

sdk.waitForClosed(new IVoidCB() {
  @Override
  public void run() {
    ... // 资源释放等清场操作
  }
});

sdk.close();

建议使用异步方式，HciSdk.waitForClosed 调用可以发生在 HciSdk.close 调用之前， 可以将清场代码放在初始化阶段，便于维护时同时修改资源分配及释放的相关代码。

HciSdk 对象在完成关闭操作后，会自动调用 HciSdk.dispose() 释放底层资源。

5.2.3 创建会话

一般使用 new 直接创建能力会话。不同的能力会话，在创建时可能需要提供不同的设置。

FreetalkStream s = new FreetalkStream(sdk, new CloudAsrConfig());

5.2.4 关闭会话

调用 HciSession.close() 方法关闭会话，该方法是一个异步方法。

同步方式

s.close();
s.waitForClosed();  // 等待会话完成关闭操作，不能在事件循环线程调用
... // 会话相关的资源释放等清场操作

异步方式

s.waitForClosed(new IVoidCB() {
  @Override
  public void run() {
    ... // 会话相关的资源释放等清场操作
  }
});

s.close();

建议使用异步方式，HciSession.waitForClosed 调用可以发生在 HciSession.close 调用之前， 可以将清场代码放在初始化阶段，便于维护时同时修改资源分配及释放的相关代码。

如果会话生命周期与 HciSdk 对象生命周期相同，用户可以不发起会话关闭操作，关闭 HciSdk 对象会间接关闭所有打开的能力会话。用户可在 HciSdk 的关闭操作回调中进行会话相关的清场操作。

如果会话生命周期比 HciSdk 对象生命周期短，建议不再使用会话后，尽早关闭会话， 以减少内存及资源占用(授权中可能会限制会话数量)。

HciSession 对象在完成关闭操作后，会自动调用 HciSession.dispose() 释放底层资源。

5.2.5 定时器

// 创建定时器
HciTimer timer = sdk.newTimer(new ITimerCB() {
  @Override
  public void run() {
    ... // 定时器被触发时执行
  }
});

// 启动定时器，1.2 秒后触发定时器，仅触发一次
timer.start(1200);

// 再次调用，将取消之前的定时设定。
// 200ms 后首次触发，然后进行间隔为 1 秒的周期性触发，直到被停止或关闭
timer.start(200, 1000);

// 停止定时器，仍可通过调用 start 启动定时器
timer.stop();

// 关闭定时器，同步操作
timer.close();

// 释放定时器资源
timer.dispose();

关闭 HciSdk 对象会间接关闭所有打开的定时器，当定时器生命周期与 HciSdk 对象生命周期相同时， 可省略定时器的关闭操作，待 HciSdk 被关闭后再释放定时器。

5.2.6 安卓音频模块

安卓平台下提供录音机和播放器。 使用录音机和播放器时，项目中需要包含 hci-android.jar, hci.jar, libhci-sys.so, libhci-common.so 中。

录音机提供回声消除和降噪处理，当硬件回声消除和降噪不可用或者效果不好时，可以尝试禁用硬件支持，使用灵云 SDK 提供的回声消除和降噪处理，可以尝试运行 audio_demo 示例程序了解更多。

还外还有 asr_demo, tts_demo, asr_tts_demo 示例供参考。
